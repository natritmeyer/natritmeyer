# Quotes

> “To me, legacy code is simply code without tests.” - Martin Fowler, Working Effectively with Legacy Code (Pearson, 2005).

> "Test code is just as important as production code. It is not a second-class citizen. It requires thought, design, and care. It must be kept as clean as production code." - Robert C. Martin, Clean Code (Prentice Hall, 2009).

> "Without \[continuous integration\], your application is broken until you prove otherwise. With \[continuous integration\], the default state of your application is working, albeit with a level of confidence that depends upon the extent of your automated test coverage." - Jez Humble & David Farley, Continuous Delivery (Addison Wesley, 2011)

> "The reasoning of mortals is worthless, and our designs are likely to fail." - Wisdom 9:14
